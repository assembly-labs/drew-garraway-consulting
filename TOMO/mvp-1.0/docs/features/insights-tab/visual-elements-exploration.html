What Changed

The content is nearly identical. What's different is the framing: a personal greeting, a milestone that uses the number as a visual anchor instead of a badge, a highlighted lead observation with supporting context below it, technique tags that feel like your training log, a focus recommendation that reads like advice from a friend, and a signoff instead of a footer disclaimer.

The Six Elements
Each has a distinct job in the debrief.
Name, ...
1. Greeting
Opens with their name + a one-line read on the week. Sets the tone. "Drew, busy week." / "Jake, still showing up." / "Marcus, honest week." This is the sports psych talking directly to them.
Lead insight
Context below
2. Highlight Card
The lead observation. Gold left border. Larger text. The one thing TOMO most wants them to know this week. Below it, a quieter context line that connects the observation to their journey.
Observation
Supporting detail...
3. Observation Card
Secondary observations. Gray left border. Quieter than the highlight. For consistency notes, technique summaries, training patterns. Can include technique tags.
Heads Up
Injury or concern...
4. Watch Card
For injuries, overtraining signals, or vulnerability patterns. Red left border + subtle red background tint. Red dot next to "Heads Up" label. Italic action line at the bottom always defers to coach/doctor.
12
rnds
Sparring data
5. Sparring Card
For sparring-specific observations. Green left border. Large number stat on the left (rounds, submissions). Body text on the right. Only for blue belts and up -- white belts don't get sparring stats quantified.
50
Milestone text
6. Milestone
No card background. Just a large gold number + text. Minimal. The number itself is the visual anchor. For session counts (10, 25, 50, 100, 200), streaks, or anniversaries. Feels earned, not gamified.
How This Changes the Prompt Output

The current edge function outputs `{ insights: [{type, title, body}], focusNext }`. To support these visual elements, the output schema would expand to:

{ greeting: "Drew, busy week.",
  highlight: { body: "...", context: "..." },
  observations: [{ label: "...", body: "...", techniques?: [...] }],
  watch?: { body: "...", action: "..." },
  sparring?: { stat: 12, unit: "rounds", body: "..." },
  milestone?: { number: 50, body: "..." },
  focusNext: "..." }

Each field is optional. The screen renders whichever elements the prompt returns. A quiet week might only have a greeting + one observation + focusNext. A big week gets all six.
