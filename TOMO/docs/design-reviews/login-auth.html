Design Notes

1 Apple Sign In is most prominent -- per Apple HIG, Sign in with Apple must be the primary option when offering third-party social login. White fill makes it visually dominant.
2 Social auth first, email second -- the "or" divider creates a visual hierarchy that nudges users toward the faster social login paths before manual entry.
3 56px input height -- generous touch targets for thumb-zone comfort. All inputs follow the same dark card-bg treatment with gold focus ring.
4 Password show/hide toggle -- reduces friction and login errors. Toggle text switches between SHOW and HIDE.
5 Legal links at bottom -- Terms and Privacy are accessible but unobtrusive. Required for App Store review.

Design Notes

1 Minimal fields -- only name, email, and password. Reduces signup friction. Additional profile details (belt rank, academy) are collected during onboarding.
2 Password requirements shown inline -- the "8+ characters" hint appears below the field rather than as a modal or toast, so the user sees the requirement before they start typing.
3 Legal consent is implicit -- per standard mobile patterns, tapping "Create Account" implies agreement. The legal text is visible but not a checkbox blocker.
4 Create Account triggers verification -- tapping the gold button in this mockup navigates to the Email Verification screen to demonstrate the flow.

Design Notes

1 Simplified layout -- only one input field. The smaller logo and clear heading make the purpose of this screen immediately obvious.
2 Back arrow navigation -- provides a clear escape hatch. Both the arrow and the "Back to Sign In" link return to the login screen.
3 Post-submit behavior -- in production, after tapping "Send Reset Link," a success message appears inline confirming the email was sent. No redirect so the user can re-send if needed.

Design Notes

1 Clear success state -- the green checkmark icon with a subtle green-tinted background circle immediately communicates that the signup step completed successfully.
2 Email displayed in bold -- showing the exact email address helps the user check the right inbox, especially if they have multiple accounts.
3 Resend link -- email delivery can be slow or caught by spam filters. The resend option reduces support requests. In production, add a cooldown timer (e.g., "Resend in 58s").
4 Supabase Auth handles verification -- Supabase sends the email with a magic link. On click, the user is verified and redirected to the onboarding flow in-app.
