TOMO DESIGN REVIEW

Phase 7: Session Logger | Generated: January 2026

Critical: Exhausted User Context

The session logger is used IMMEDIATELY post-training. Users are physically exhausted, cognitively fatigued (20-40% reduced decision-making), sweaty, and have a 90-second tolerance window. Design must prioritize: large touch targets (56-80px), one question at a time, voice-first input, and forgiving natural language parsing.

Logger Entry Screen Notes

  • Full-screen overlay (blocks distractions)
  • Close button (X) in top-right corner
  • Core fields visible without scrolling
  • Large microphone button for voice input
  • Text fallback always available

Logger Flow Phases

LOG-ENTRY: Entry Screen

  • Training type selector (Gi / No-Gi / Open Mat)
  • Duration picker (quick presets: 60, 90, 120 min)
  • Sparring toggle (Yes/No)
  • Large record button for voice input

LOG-RECORD: Recording State

  • Pulsing microphone animation
  • Recording duration counter
  • Voice prompts guide input
  • "Just talk naturally - what did you work on?"
  • Stop button to end recording

LOG-PROCESS: Processing State

  • Spinner animation
  • "Extracting your training details..."
  • AI parses voice transcript
  • Extracts: techniques, partners, outcomes

LOG-REVIEW: Review Screen

  • Confirmation chips for extracted data
  • Tap to edit any field
  • Add/remove items easily
  • "Log Session" final confirmation button

LOG-SUCCESS: Success State

  • Checkmark animation
  • Belt-adaptive post-session message
  • "Nice work! Session logged."
  • Option to view in journal or close

Form Fields Reference

LOG-TYPE Training Type: Gi / No-Gi / Open Mat
LOG-DURATION Duration: Time picker with presets
LOG-SPAR Sparring: Yes/No toggle
LOG-VOICE Voice Input: Microphone button
LOG-TEXT Text Fallback: Textarea input

Post-Session Messages (Belt-Adaptive)

Belt-Specific Tone

  • LOG-MSG-WHITE: Encouragement, consistency focus - "Every session counts. You showed up today."
  • LOG-MSG-BLUE: Progress tracking, plateau acknowledgment - "Keep building. Your game is evolving."
  • LOG-MSG-PURPLE: Depth, teaching perspective - "Refining the details. The mat remembers."
  • LOG-MSG-BROWN: Refinement, legacy thinking - "Polishing mastery. Leading by example."

Session Logger Design Mandates

Non-Negotiable Requirements

  • Touch targets: 56-80px minimum for primary actions
  • One question at a time - don't overwhelm
  • Voice-first - typing is a fallback, not primary
  • Skip always available - never force non-critical input
  • Instant feedback - show what was captured immediately
  • Generous defaults - pre-fill when possible
  • Forgiving input - parse natural language, don't require formats
  • 90-second target - entire flow should complete in under 90s
