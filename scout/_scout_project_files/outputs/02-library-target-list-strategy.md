# Scout & Finch: Target Library Prioritization Strategy

## Executive Summary

This document outlines a systematic approach to identifying, prioritizing, and engaging with US public libraries for Scout & Finch adoption. Based on our context research and upcoming Tredyffrin demo, we're targeting libraries using Polaris, Koha, and Sierra ILS systems, with initial focus on Pennsylvania and Delaware (local advantage) plus high-opportunity states.

**Key Strategy Points:**
- Leverage local Philadelphia/Delaware presence for warm introductions
- Prioritize Polaris, Koha, and Sierra ILS (Tredyffrin uses Vega/Sierra)
- Focus on 7 states including PA/DE for local advantage
- Align outreach timing with library fiscal year cycles (January-March is prime)
- Build from individual libraries to consortiums for scale

---

## Segmentation Criteria

### Tier 1: Ideal Beta Candidates (30-50 libraries)

**Characteristics:**
- **Size:** 25K-100K patrons served (sweet spot for budget and need)
- **ILS:** Currently using Polaris, Koha, or Sierra (proven integration paths)
- **Geography:** Located in innovation-friendly states, prioritizing PA/DE region
- **Budget:** $15K-$35K available for new software tools
- **Innovation Indicators:**
  - Recent technology adoptions (self-checkout, maker spaces, digital services)
  - Active social media presence
  - Participation in state/national innovation programs
  - Library director speaks at conferences
  - Recent grant recipients for technology projects

**Why This Tier First:**
- Budget authority at director level (faster decisions)
- Sophisticated enough to be early adopters
- Not so large that procurement is byzantine
- Staff size creates real pain points Finch solves
- Influence on peer libraries in region
- Likely to provide actionable feedback

**Outreach Priority:** Immediate (December 2024 - February 2025)

### Tier 2: Early Adopter Prospects (100-150 libraries)

**Characteristics:**
- **Size:** 15K-75K patrons (broader range)
- **ILS:** Polaris, Koha, OR Sierra (Phase 2 integration)
- **Geography:** Secondary markets in target states
- **Budget:** $10K-$25K for software
- **Innovation Indicators:**
  - Follow industry trends but not first movers
  - Look to Tier 1 libraries for validation
  - Active in state library associations
  - Have dedicated IT or tech services librarian

**Why This Tier Second:**
- Need peer validation from Tier 1
- Larger addressable market
- May need more hand-holding
- Good for scaling after product refinement
- Can approach with case studies from Tier 1

**Outreach Priority:** March-May 2025 (after Tier 1 pilots show results)

### Tier 3: Consortium & System Opportunities (10-15 consortia)

**Characteristics:**
- **Size:** 5-50 member libraries per consortium
- **Decision-Making:** Centralized or committee-based
- **Geography:** State or regional coverage
- **Budget:** Negotiated volume pricing
- **Examples:**
  - Massachusetts Library System (MLS)
  - Colorado Library Consortium (CLiC)
  - North Carolina LIVE
  - Reaching Across Illinois Library System (RAILS)
  - PALCI (Pennsylvania Academic Library Consortium)

**Why Consortiums Matter:**
- One decision impacts dozens of libraries
- Volume pricing justification
- Shared implementation costs
- Peer pressure for adoption
- Simplified procurement

**Outreach Priority:** June-August 2025 (with proven results from Tiers 1-2)

### Tier 4: Large Urban Systems (Future)

**Characteristics:**
- **Size:** 250K+ patrons
- **Examples:** NYPL, Chicago Public, LA Public
- **Budget:** $75K+ potential
- **Challenges:** Complex procurement, multiple stakeholders, long sales cycles

**Why Not First:**
- 6-18 month sales cycles
- RFP requirements
- Multiple decision layers
- Risk-averse
- Better approached with proven track record

**Outreach Priority:** Year 2 (2026) after establishing market presence

## Research Sources for List Building

### Primary Data Sources

**1. Library Technology Guides (Marshall Breeding)**
- URL: librarytechnology.org
- Contains: ILS usage by library, budget data, contact info
- Cost: Free
- Use: Identify Polaris/Koha libraries in target states

**2. State Library Directories**
- Each state maintains directory with contacts
- Usually includes: Director names, emails, phone
- Cost: Free
- Examples:
  - California: directory.library.ca.gov
  - Massachusetts: libraries.state.ma.us
  - Colorado: coloradovirtuallibrary.org

**3. LinkedIn Sales Navigator**
- Filter by: Title, library name, location
- Find: Directors, tech librarians, decision makers
- Cost: $79/month
- ROI: Direct access to decision makers

**4. IMLS Public Library Survey Data**
- URL: imls.gov/research-evaluation/data-collection/public-libraries-survey
- Contains: Budget data, staff size, population served
- Cost: Free
- Use: Qualify libraries by size and budget

### Secondary Data Sources

**5. Library Websites**
- Staff directories
- Board meeting minutes (budget discussions)
- Strategic plans (technology priorities)
- News/announcements (grants received)

**6. State Library Association Member Lists**
- Often available to members
- Conference attendee lists
- Committee memberships
- Cost: $200-$500/year membership

**7. Grant Award Announcements**
- IMLS grant recipients
- State LSTA awards
- Shows innovation appetite and funding

### Email Finding Tools

**Recommended Tools:**
- **Hunter.io:** $49-$149/month, best for bulk finding
- **Apollo.io:** $49-$149/month, includes CRM features
- **Clearbit:** $99+/month, most accurate but expensive
- **RocketReach:** $39-$99/month, good for individuals

**Free Alternatives:**
- Google advanced search operators
- Library website staff pages
- State government employee directories
- LinkedIn manual outreach

## Long List Template & Instructions

### CSV Structure

The template CSV (see accompanying file: `02-library-longlist-template.csv`) contains:

**Column Definitions:**

1. **Library_Name:** Official library name
2. **City:** City location
3. **State:** Two-letter state code
4. **Service_Population:** Number of patrons served
5. **Annual_Budget:** Total operating budget (if available)
6. **ILS_System:** Current ILS (Polaris, Koha, Sierra, etc.)
7. **Director_Name:** Library Director full name
8. **Director_Email:** Direct email if available
9. **Director_LinkedIn:** LinkedIn profile URL
10. **Tech_Librarian_Name:** Technology/Systems Librarian
11. **Tech_Librarian_Email:** Direct email
12. **Website:** Library website URL
13. **Tier:** 1, 2, or 3 based on criteria
14. **Innovation_Signals:** Recent tech adoptions, grants, initiatives
15. **Fiscal_Year_End:** Month of FY end (typically June)
16. **LSTA_Eligible:** Yes/No for grant funding
17. **Consortium_Member:** Name of consortium if applicable
18. **Priority_Score:** 1-10 (10 highest)
19. **Notes:** Any relevant context

### Building Your Long List

**Step 1: ILS Filtering**
- Start with Library Technology Guides
- Filter for Polaris (704 libraries)
- Filter for Koha (1,653 libraries)
- Export to spreadsheet

**Step 2: Geographic Filtering**
- Filter for 5 primary states
- Remove libraries under 10K population
- Remove libraries over 250K (save for Tier 4)

**Step 3: Enhancement**
- Add director names from state directories
- Find emails using Hunter.io
- Add LinkedIn profiles
- Note recent innovations from websites

**Step 4: Scoring**
- Assign tier based on criteria
- Score 1-10 based on:
  - ILS match (Polaris = 3, Koha = 2)
  - Population fit (25K-100K = 3)
  - Innovation signals (0-2)
  - State priority (Primary = 2)
