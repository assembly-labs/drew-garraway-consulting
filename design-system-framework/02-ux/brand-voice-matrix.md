## Applying This Matrix

### For UX Copy
- Reference tone spectrum based on context
- Use approved terminology
- Check writing samples for patterns
- Avoid banned words/phrases

### For Marketing Copy
- Lean into personality traits
- Can be slightly more enthusiastic than UX copy
- Reference voice principles

### For Support/Documentation
- Lean into supportive tone
- Prioritize clarity over personality
- Technical level can flex up

### For Error Messages
- Always serious tone
- Never blame user
- Always provide next step
- No humor, no emoji

Before shipping copy, verify:
